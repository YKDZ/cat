# Vue i18n 规范

## 正面限制

1. **所有用户可见文本都必须走 i18n。** 模板和 `<script setup>` 中的展示文案统一通过 `vue-i18n` 输出。
2. **在组件顶层初始化 `useI18n()`。** 需要本地化的 Vue 组件应在 `<script setup lang="ts">` 中导入并初始化 `t`。
3. **默认使用中文自然语言作为 key。** 除非某个模块已经明确要求 dot-notation key，否则直接使用最终展示的中文文案作为翻译 key。
4. **动态值使用命名插值。** 变量统一写成 `{name}` 占位，并通过 `t()` 第二个参数传入对象。
5. **属性和组件 props 也要本地化。** `title`、`placeholder`、`alt` 等属性，以及透传给组件的显示型 prop，都应通过 `:prop="t(...)"` 绑定。
6. **缺省文案也要可翻译。** 处理 `null` / `undefined` / 空值时，回退文本同样使用 `t()`。

## 负面限制

1. **不要硬编码用户可见字符串。** 无论是中文、英文，还是拼接后的字串，都不要直接写死在模板或脚本里。
2. **不要随手引入抽象 key。** 如果当前模块没有既定 key 体系，就不要把 `上一页` 写成 `pagination.prev`。
3. **不要直接字符串拼接动态值。** 统一使用命名插值，避免文案顺序在不同语言下失控。
4. **不要忽略属性本地化。** `title="..."`、`placeholder="..."` 这类原始字符串同样属于硬编码。

## 例子

### 推荐：组件内初始化并使用 `t()`

```ts
import { useI18n } from "vue-i18n";

const { t } = useI18n();
```

### 推荐：使用自然语言 key 与命名插值

```vue
{{ t("{amount} 个术语", { amount: concept.termCount }) }}
{{
  t("显示 {from} - {to} 条，共 {total} 条", {
    from: pageIndex * pageSize + 1,
    to: Math.min((pageIndex + 1) * pageSize, total),
    total,
  })
}}
```

### 推荐：属性与嵌套翻译

```vue
<button
  :title="
    t('类型：{type} | 状态：{status}', {
      type: t(termItem.type),
      status: t(termItem.status),
    })
  "
/>
```

### 推荐：本地化回退文案

```vue
{{ concept.definition || t("—") }} {{ concept.subject || t("（未命名）") }}
```

### 避免：硬编码与抽象 key 混用

```vue
<!-- ❌ Avoid -->
<button title="上一页">{{ t("pagination.prev") }}</button>
```
